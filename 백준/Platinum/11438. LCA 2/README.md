### 문제 설명

<p>N(2 ≤ N ≤ 100,000)개의 정점으로 이루어진 트리가 주어진다. 트리의 각 정점은 1번부터 N번까지 번호가 매겨져 있으며, 루트는 1번이다.</p>

<p>두 노드의 쌍 M(1 ≤ M ≤ 100,000)개가 주어졌을 때, 두 노드의 가장 가까운 공통 조상이 몇 번인지 출력한다.</p>

### 입력 

 <p>첫째 줄에 노드의 개수 N이 주어지고, 다음 N-1개 줄에는 트리 상에서 연결된 두 정점이 주어진다. 그 다음 줄에는 가장 가까운 공통 조상을 알고싶은 쌍의 개수 M이 주어지고, 다음 M개 줄에는 정점 쌍이 주어진다.</p>

### 출력 

 <p>M개의 줄에 차례대로 입력받은 두 정점의 가장 가까운 공통 조상을 출력한다.</p>

### 코드 리뷰
이 코드는 트리 구조에서 두 노드의 가장 가까운 공통 조상(LCA, Lowest Common Ancestor)을 효율적으로 찾기 위한 알고리즘을 구현한 것입니다. 코드는 크게 세 부분으로 나눌 수 있습니다.

```cpp
// DFS: depth, parent[0] 계산
void dfs(int cur, int par) { ... }
```
**설명 1**: `dfs` 함수는 깊이 우선 탐색(DFS)을 사용하여 각 노드의 깊이와 0번째 조상(부모 노드)을 계산합니다. `cur`는 현재 노드, `par`는 부모 노드를 나타내며, 각 노드를 탐색하면서 깊이를 설정하고 부모 정보를 저장합니다.

```cpp
void buildParent() { ... }
```
**설명 2**: `buildParent` 함수는 `parent` 배열을 채우기 위한 함수로, 각 노드에 대해 2의 거듭 제곱(k) 번째 조상을 계산합니다. 이는 LCA 쿼리를 O(log N) 시간 내에 처리할 수 있도록 준비하는 과정입니다.

```cpp
int lca(int a, int b) { ... }
```
**설명 3**: `lca` 함수는 두 노드 `a`와 `b`의 LCA를 찾는 함수입니다. 먼저 두 노드의 깊이를 맞춘 후, 두 노드의 조상이 다를 때까지 올라가며 공통 조상을 찾습니다.

### 시간/공간 복잡도 분석
- **시간 복잡도**:
  - DFS는 O(N) 시간 복잡도를 가집니다. 
  - `buildParent` 함수는 O(N log N) 시간 복잡도를 가지며, 각 노드에 대해 O(log N)만큼의 작업을 수행합니다.
  - LCA 쿼리는 O(log N) 시간 복잡도를 가집니다. 따라서 Q개의 쿼리를 처리하는 데 O(Q log N) 시간이 소요됩니다.
  - 전체적으로 시간 복잡도는 O(N + Q log N)입니다.

- **공간 복잡도**:
  - `tree`는 O(N) 공간을 사용하고, `parent` 배열은 O(N log N) 공간을 사용합니다.
  - `depth` 배열은 O(N) 공간을 사용합니다.
  - 전체적으로 공간 복잡도는 O(N log N)입니다.

### 코드 최적화 가능성 및 개선 제안
- **최적화 가능성**: 현재 코드 구조는 LCA를 찾기 위한 일반적인 최적화 기법인 희소 배열을 사용하고 있어 효율적입니다. 그러나 `buildParent` 함수를 호출하기 전에 `parent` 배열을 초기화하는 코드를 추가하는 것이 좋습니다. 현재는 `parent` 배열이 초기화되지 않고 사용되므로 예기치 않은 동작을 초래할 수 있습니다.

- **개선 제안사항**: 
  - `dfs` 함수 내에서 루프를 통해 `parent` 배열을 채우는 과정에서 `parent[0][cur] = par;`를 호출하고 있는데, 이는 DFS의 첫 단계에서 부모를 설정하므로 코드 가독성을 위해 루프 바깥에서 설정할 수 있습니다.
  - `lca` 함수의 두 번째 부분에서 `if (parent[k][a] != parent[k][b])` 비교를 하면서 `parent[k][a]`와 `parent[k][b]`를 가져오는 과정에서 반복적으로 메모리 접근이 발생하므로, 이를 한 번에 저장해두고 비교하는 방법으로 최적화할 수 있습니다.

- **가독성 및 유지보수성**: 전체적으로 코드가 명료하고 각 함수의 기능이 잘 분리되어 있어 가독성이 뛰어납니다. 적절한 주석이 포함되어 있어 유지보수가 용이합니다. 그러나 변수명 및 주석을 약간 더 명확하게 하여 가독성을 높일 수 있습니다 (예: `cur` 대신 `currentNode`, `par` 대신 `parentNode` 등).
