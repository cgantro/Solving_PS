### 문제 설명

<p>루트가 있는 트리(rooted tree)가 주어지고, 그 트리 상의 두 정점이 주어질 때 그들의 가장 가까운 공통 조상(Nearest Common Ancestor)은 다음과 같이 정의됩니다.</p>

<ul>
	<li>두 노드의 가장 가까운 공통 조상은, 두 노드를 모두 자손으로 가지면서 깊이가 가장 깊은(즉 두 노드에 가장 가까운) 노드를 말합니다.</li>
</ul>

<p>예를 들어  15와 11를 모두 자손으로 갖는 노드는 4와 8이 있지만, 그 중 깊이가 가장 깊은(15와 11에 가장 가까운) 노드는 4 이므로 가장 가까운 공통 조상은 4가 됩니다.</p>

<p>루트가 있는 트리가 주어지고, 두 노드가 주어질 때 그 두 노드의 가장 가까운 공통 조상을 찾는 프로그램을 작성하세요</p>

### 입력 

 <p>첫 줄에 테스트 케이스의 개수 T가 주어집니다.</p>

<p>각 테스트 케이스마다, 첫째 줄에 트리를 구성하는 노드의 수 N이 주어집니다. (2 ≤ N ≤ 10,000)</p>

<p>그리고 그 다음 N-1개의 줄에 트리를 구성하는 간선 정보가 주어집니다. 한 간선 당 한 줄에 두 개의 숫자 A B 가 순서대로 주어지는데, 이는 A가 B의 부모라는 뜻입니다. (당연히 정점이 N개인 트리는 항상 N-1개의 간선으로 이루어집니다!) A와 B는 1 이상 N 이하의 정수로 이름 붙여집니다.</p>

<p>테스트 케이스의 마지막 줄에 가장 가까운 공통 조상을 구할 두 노드가 주어집니다.</p>

### 출력 

 <p>각 테스트 케이스 별로, 첫 줄에 입력에서 주어진 두 노드의 가장 가까운 공통 조상을 출력합니다.</p>

### 코드 리뷰
```cpp
#include<iostream>
#include<vector>
#include<unordered_set>
using namespace std;
int N;

const int ROOT = 0;
vector<int> parent;
unordered_set<int> _parents;
int main(){	
	ios_base::sync_with_stdio(false);
	cin.tie(nullptr);
	cout.tie(nullptr);

	int T; cin >> T;
	while(T--){
		cin >> N; parent.assign(N+1,0);
		for(int i=0; i<N-1; i++){
			int A,B; cin >> A >> B;
			parent[B] = A;
		}
		int n1,n2; // node1, node2
		cin >> n1 >> n2;
		_parents.clear();
		int ans = 0;
		while(true){
			_parents.insert(n1);
			int pr = parent[n1];
			if(pr == 0) break;
			n1 = pr;
		}

		while(true){
			if(_parents.count(n2)) {
				ans = n2;
				break;
			}
			int pr = parent[n2];
			n2 = pr;
		}

		cout << ans << '\n';
	}
}
```
이 코드는 주어진 트리에서 두 노드의 가장 가까운 공통 조상(Nearest Common Ancestor, LCA)을 찾기 위한 알고리즘입니다. 각 테스트 케이스마다 트리를 구성하는 노드 수 N과 N-1개의 간선 정보를 입력받고, 두 노드 n1과 n2의 LCA를 출력합니다.

- **코드 블록 1**: 트리의 부모 관계를 읽어들여 `parent` 벡터에 저장합니다. 
- **설명 1**: 이 부분에서는 N-1개의 간선 정보를 입력받아 각 노드의 부모를 기록합니다. `parent[B] = A`를 통해 B의 부모가 A임을 설정하여 트리 구조를 만듭니다.

- **코드 블록 2**: 두 노드 n1과 n2의 LCA를 찾기 위해 n1의 모든 조상을 `_parents` 집합에 저장합니다. 
- **설명 2**: 첫 번째 while 루프를 통해 n1의 모든 조상을 `_parents`에 추가하고, n1이 루트에 도달하면 종료합니다. 두 번째 while 루프에서는 n2의 조상을 차례로 확인하여 `_parents`에 존재하는지 검사하고, 존재하면 해당 노드를 LCA로 설정합니다.

### 시간/공간 복잡도 분석
- **시간 복잡도**: 각 테스트 케이스에서 트리의 모든 간선 정보를 읽고, 두 노드의 조상을 찾는 데 O(N) 시간이 소요됩니다. N개의 노드가 있을 때, 두 개의 while 루프 각각에서 최대 N/2 번의 반복이 이루어질 수 있으므로 전체 시간 복잡도는 O(N)입니다. 테스트 케이스 T가 주어지면 최악의 경우 O(T * N)입니다.
- **공간 복잡도**: 부모 정보를 저장하는 `parent` 벡터와 조상을 저장하기 위한 `_parents` 집합의 공간이 필요합니다. 이 두 개의 자료구조를 합치면 O(N)의 공간 복잡도를 가집니다.

### 코드 최적화 가능성 및 개선 제안
- **최적화 가능성**: 현재 알고리즘은 간단하면서도 효율적입니다. 다만, 각 노드의 깊이 정보를 미리 계산하여 LCA를 찾는 과정에서 더 빠른 방법(예: Binary Lifting 기법)을 사용할 수 있습니다. 이는 O(log N) 시간 복잡도로 LCA를 찾을 수 있게 해 줄 수 있습니다. 
- **개선 제안사항**: 
  - `unordered_set` 대신 `unordered_map`을 사용해 각 노드의 조상을 저장하고 탐색하는 방법을 고려할 수 있습니다. 
  - 코드 가독성을 위해 상수명과 변수명을 더 직관적으로 변경하는 것도 좋습니다 (예: `parent`를 `parents`로 변경). 
  - 주석을 추가하여 각 단계의 목적을 명확히 하는 것도 가독성과 유지보수성을 높일 수 있습니다.
- **가독성과 유지보수성**: 현재 코드 구조는 명확하게 목적을 가지고 있지만, 루프와 조건문에 대한 설명이 부족하여 초보자가 이해하기 어려울 수 있습니다. 주석을 추가하고, 함수로 분리하면 유지보수성이 향상될 것입니다.
